## Описание системы

Система реализует автоматизированное управление парковкой на основе мультиагентного pipeline. Входящий запрос последовательно обрабатывается четырьмя независимыми агентами на Go, общающимися через брокер сообщений NATS.

| Агент     | Канал NATS               | Роль в системе                                    |
|-----------|--------------------------|---------------------------------------------------|
| `search`  | `parking.search`         | Поиск свободных мест в заданной зоне              |
| `booking` | `parking.book`           | Бронирование места, хранение состояния в Redis    |
| `access`  | `parking.access`         | Контроль въезда/выезда по booking_id              |
| `payment` | `parking.payment`        | Расчёт и проведение оплаты                        |

**Ключевые особенности:**

- **4 агента на Go** — единый модуль (`parking-system`), агенты запускаются как отдельные команды через `go run ./agents/<name>`
- **Pipeline оркестрации** — поиск → бронь → доступ → оплата, управляется Python-оркестратором через FastAPI
- **Агент с состоянием** — агент `booking` сохраняет брони в Redis (`SET booking:{id}`), восстанавливает состояние при перезапуске
- **Аукционное распределение** — несколько реплик `search`-агента конкурируют за задачу через `auction.search.request`, оркестратор выбирает с минимальной ценой
- **Динамическое масштабирование** — `scaler/` мониторит длину очереди NATS, при превышении порога запускает дополнительные контейнеры через Docker Socket Proxy
- **LLM-агент** — анализирует текстовые запросы через Ollama (llama3.2:1b), результат кэшируется в Redis
- **Распределённая трассировка** — OpenTelemetry v1.21 + Jaeger для визуализации полного пути задачи
- **Веб-дашборд** — Streamlit-панель со статусом агентов, историей броней и ручным запуском

## Pipeline

```
HTTP-запрос
     │
     ▼
Оркестратор (FastAPI + Python)
     │
     ├─→ parking.search  → [свободные места]
     │
     ├─→ parking.book    → [booking_id]
     │
     ├─→ parking.access  → [въезд подтверждён]
     │
     └─→ parking.payment → [оплата проведена]
                               │
                               ▼
                          HTTP-ответ
```

## Быстрый старт

### 1. Запуск инфраструктуры

```bash
docker-compose up -d
```

### 2. Загрузка LLM-модели

```bash
docker exec -it ollama ollama pull llama3.2:1b
```

### 3. Установка зависимостей оркестратора

```bash
cd orchestrator
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 4. Запуск дашборда

```bash
cd webui
pip install -r requirements.txt
streamlit run app.py
```

### 5. Тестовый запрос pipeline

```bash
curl -X POST http://localhost:8000/pipeline \
  -H "Content-Type: application/json" \
  -d '{"zone": "A", "car_number": "A123BC", "hours": 2}'
```
